
Week 1: Data Acquisition, Cleaning, and Feature Engineering

The first week focused on setting up the GitHub repository and importing the travel dataset into the Python environment. The dataset was carefully examined to understand its structure, data types, and quality. Data preprocessing involved handling missing values, removing duplicate records, correcting inconsistent formats, and validating the dataset for accurate analysis. Outlier detection was performed on accommodation and transportation costs to improve data quality. Feature engineering techniques were applied to create new attributes such as Total Cost, Cost Per Day, Travel Month, Travel Year, and Age Group, which enhanced the dataset for further analysis and predictive modeling.

Week 2: Exploratory Data Analysis (EDA) and Statistical Testing

During the second week, exploratory data analysis was carried out using Pandas, Matplotlib, and Seaborn to understand travel trends and customer behavior. Various visualizations, including histograms, bar charts, box plots, pie charts, and correlation heatmaps, were developed to identify patterns in travel expenses, accommodation preferences, transportation methods, and destination popularity. Statistical summaries and correlation analysis were performed to examine relationships between numerical variables such as travel duration, accommodation cost, transportation cost, and total travel cost. Key findings and business insights were documented using Markdown cells in Jupyter Notebook.

Week 3: Baseline Predictive Modeling

The third week concentrated on building a baseline predictive model using Scikit-Learn. Data preprocessing included label encoding for categorical variables and splitting the dataset into training and testing sets using an 80:20 ratio. A Logistic Regression model was developed to classify travel records into High Cost and Low Cost categories based on selected features. Model performance was evaluated using Accuracy, Precision, Recall, F1-Score, ROC-AUC Curve, and Confusion Matrix. The predictive model provided useful insights for travel cost estimation and supported business decisions related to customer retention and dynamic pricing strategies.

Week 4: Dashboard Development and Documentation

The final week focused on transforming analytical results into an interactive Power BI Dashboard for business users. The dashboard included KPI Cards, Line Chart, Clustered Column Chart, Donut Chart, Clustered Bar Chart, 100% Stacked Bar Chart, Map Visualization, and Interactive Slicers to present travel trends, destination analysis, traveler demographics, and cost comparisons. Business insights and recommendations were developed to support travel planning and pricing decisions. The GitHub repository was finalized with complete project documentation, Jupyter Notebooks, dashboard screenshots, project structure, and a comprehensive README.md, ensuring the project was well-organized and reproducible.
